## Issues Encountered & Resolutions

### Issue 1: Module/Package Naming Conflict
**Description:** yaml_loader.py and validator.py had broken imports due to `src/selectors/models.py` conflicting with `src/selectors/models/` package.

**Resolution:** Used `importlib.util` to directly import from models.py file, renamed ValidationResult to avoid naming conflict.

**Impact:** Fixed across 2 files, affected Story 1-1 and carried forward.

### Issue 2: Deprecated datetime.utcnow()
**Description:** Multiple files used deprecated `datetime.utcnow()` which produces naive datetime objects.

**Resolution:** Replaced with `datetime.now(timezone.utc)` in:
- src/selectors/fallback/chain.py (9 occurrences)
- src/selectors/fallback/models.py (1 occurrence)
- tests/selectors/fallback/test_chain.py (4 occurrences)

**Impact:** Fixed in Story 1-2, but some remaining in Story 1-4 (pending).

### Issue 3: Test Quality - Attribute Existence vs Actual Execution
**Description:** Initial tests only verified methods exist via hasattr checks, didn't verify actual selector execution.

**Resolution:** Created TestPrimarySelectorExecution class with tests that verify actual execution against mock DOM using Playwright page mocks.

**Impact:** 12 new tests added, all passing. Pattern established for future stories.

## Lessons Learned

### What Worked Well
1. **Decorator Pattern** - @with_fallback decorator provides clean, declarative fallback chains
2. **Structured Logging** - Using structlog with correlation IDs enables excellent debugging
3. **Test-First Approach** - Upgrading tests to verify actual execution caught real issues
4. **Incremental Stories** - Linear dependency (1-1 → 1-2 → 1-3 → 1-4) worked well

### What Could Be Improved
1. **Earlier Import Testing** - Module import conflicts discovered late; should verify early
2. **Deprecation Scanning** - Automated scanning for deprecated patterns (e.g., utcnow()) would help
3. **AC Validation** - Story 1-4 had AC mismatch (2 entries vs 1); better AC review needed
4. **Cross-Story Debt Tracking** - Some debt from earlier stories wasn't fully resolved

### Patterns to Reuse
- Use `importlib.util` for resolving module/package name conflicts
- Tests should verify actual execution, not just attribute existence
- Decorator pattern for cross-cutting concerns (fallback, logging)
- Structured logging with correlation IDs for debugging
