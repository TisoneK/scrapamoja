2. **Live DOM extraction (HIGH):** 70 events with garbled names, 0 markets, 0 scores. Live page renders differently (Vue components for in-play state) and selectors don't match.
3. **Team field names (LOW):** Stored as `home`/`away` instead of `home_team`/`away_team`. Consistent across all events but differs from what quality scripts check for.
4. **H2H field mapping (LOW):** `H2HData` model lacks `total_matches`/`home_wins`/`away_wins` computed fields that consumers might expect. Raw `game_shorts` array has all data (26 entries) ‚Äî consumers must compute summaries.
