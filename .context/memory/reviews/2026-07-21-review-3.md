3. **Change-only dedup** (`9b72f0e`) — odds/state/period rows insert only on change
   (identical poll → 0 rows; one price move → 1). Records movement, not heartbeats.
4. **Interval poller** (`a1b9e08`) — `betb2b poll` loops scrape+persist; pure,
   injected-clock `poll_loop`.
5. **Browser-free HTML harvest** (`6150884`) — the DOM virtualizes (~10 of a bigger

   `_harvest_events` (httpx GET page → GetGameZip each) is now the PRIMARY discovery
   path (`_discover_events`, flag `html_harvest`); DOM is fallback. Answers "use the
   endpoint, not the UI".
6. **Multi-skin** (`9d64f6e`) — comma-list / `--all-skins` on `scrape` and `poll`;
   all skins persist to one store → `cross_skin_odds` "just works".

## Engine integration path (investigated, not built)

`scorewise-engine` (sibling repo) is a basketball totals+winner prediction pipeline;
its `POST /api/ingest` accepts `IngestRequest{source, scraped_at, matches:[PredictRequest]}`.
`PredictRequest` = match_id + teams + `odds{match_total(.5), over/under, home/away}` +
`h2h_matches[]`. The betb2b `Event` maps directly (event_id, home/away, Total
Over/Under market → match_total+odds, To Win Match → home/away odds, h2h_data.game_shorts
→ h2h_matches). Next: an exporter + ingest client in the scraper (repos stay isolated).
