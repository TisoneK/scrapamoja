## Findings & fixes

### F1 — GetGameZip enrichment skipped every DOM event (HIGH, fixed `99be8ac`)
`scraper._enrich_dom_events_with_odds` guarded `if e.markets: skip`. But the
DOM extractor always attaches a shallow 1-market stub ("To Win Match"/"1x2")
when it finds odds, so **every** DOM event was truthy-`markets` and skipped →
0 GetGameZip fetched in Session 24 despite 28 events. Fix: skip only when the
event already has a **deep** tree (`len(e.markets) > 1`). 0/1-market DOM stubs
now get the full `E[]/AE[]` tree. Verified on a real capture: 1 stub → 10
markets / 33 selections (prematch); live IDs → 40 / 9 / 7 markets.
Added `test_betb2b_enrichment.py` (6 tests) + a real GetGameZip fixture.

### F2 — Garbled live team names passed the guard (HIGH, fixed `58f9a46`)
`"Ajax Olympiacos Piraeus 0000-Ajax Olympiacus Piraeus 0000"` passed
`_is_plausible_team_name` (≤80 chars; `^0000$` only matched the whole value).
Fix: reject the `0000` placeholder token **anywhere**, and reject
concatenation artifacts via a repeated-8+-char-chunk duplication detector.
0 false-rejects across 15 real names. Added `test_betb2b_dom_guard.py` (20
tests). Note: on a **fresh** capture the current selectors already extract
clean names — the Session 24 garble was likely a loading-state snapshot.

### F3 — Live in-play scores not extracted (HIGH, fixed `26b08d5`)
The in-play grid renders the running total as two adjacent
`.ui-game-scores__num` spans inside `.ui-game-scores__item--total`; the old
score selectors didn't match it → `score None-None`. Fix: add
`.ui-game-scores__item--total .ui-game-scores__num` as the primary
`team_scores` selector and teach `_score_pair` to accept the whitespace-joined
pair "46 57". Verified: 10 live events, 100% parsed scores. Prematch
unaffected.

### F4 — Fixed-settle render is fragile through a slow proxy (MED, fixed `d173c6a`)
`render_dom_events` used `wait_until="commit"` + a fixed `sleep(settle)`, then
extracted. Through the slow tunnel the Vue grid hadn't rendered → the
integrated run got **0 raw rows** despite a live card a standalone probe read
as 10 games. Fix: after the settle, actively `wait_for_selector` on the
game-row selector (`grid_wait_ms`, default 20s), best-effort. Makes
live/prematch DOM robust to render-speed variance.

### F5 — Proxy verification aborted the scrape on one transient error (MED, fixed `7f59edc`)
`_verify_proxy_country` did a single `verify_proxy` call; one `ReadError` on
the pre-flight egress check aborted the whole multi-minute scrape even though
the tunnel answered seconds later (two live runs died this way). Fix: retry
transient reachability failures with linear backoff (`proxy_verify_attempts=3`,
`proxy_verify_backoff=3s`); a definitive country mismatch still fails fast.
+4 tests.
